Overview

The entire MVP, on one page.

This walkthrough shows the full folio.e8e product surface in design-quality detail — from first-run onboarding to the retrospective after a trade. Every screen is built from the same component vocabulary, the same type system, and the same color semantics, so the product reads as a coherent whole rather than a collection of features.

Use it as a shared reference. Designers will extract tokens and patterns into a real Figma system. Engineers will use it to understand what they are building toward. Stakeholders will see the finished shape before a line of production code is written.

Act I — First run
Meet the agent, connect the broker.
5 screens · ~90 seconds

Act V — Type & system
The vocabulary behind everything above.
Design tokens · principles

Every screen above is composed from the same three fonts, the same six colors, and the same handful of component patterns. This is what keeps generative UI coherent: the LLM can arrange many layouts, but it cannot invent new grammar.

Display · Fraunces
AaBb
A serif with warmth.
Headlines, portfolio values, decision cards, retrospective moments. Italic in amber marks agent voice. Optical sizing at 144 for display, 9 for footnotes.
Technical · JetBrains Mono
$104,218.46
DELTA · −0.18 · DTE 21 · MAT 0.72
Numbers, tickers, constraint chips, timestamps, trace IDs. Never in flowing copy. Signals “this is a real system.”
Voice · Inter Tight
The clear narrator.
Body copy, explanations, UI labels. Inter Tight for clarity at small sizes and mechanical feel that pairs with both serif and mono.
Ink Bg#0F100D
Surface#1C1D19
Amber#D4A25E
Teal#6FB3A4
Sage#8EA87C
Cream#F6EFE0

Amber — agent voice, pending actions, urgent moments. Used sparingly to keep signal strong. Teal — confirmed states, fills, closed-and-won positions. Sage — healthy systems, passed constraints, quiet profits. Rose — for halts and destructive actions, used with restraint. Cream — reserved for future editorial surfaces and educational moments. Dominant surfaces are warm off-blacks, never pure black or pure white.

Design charter · five rules
1
Editorial over dashboard.

Every screen reads like a considered page, not a grid of widgets.

2
Amber is sacred.

Only the agent's voice gets amber. Never decoratively.

3
Show the reasoning.

Every action includes why, signals, constraint checks. Visible, always.

4
Numbers are monospaced.

Prices, tickers, Greeks, timestamps — always JetBrains Mono.

5
No green-red cliché.

P&L uses sage and rose — meaning retained, cliché avoided.

Quiet. Intelligent. Trusted.
