02 — THE WHEEL, EXPLAINED
What the agent actually does.
4-step cycle · repeats indefinitely

The Wheel is a conservative options strategy. Sell a put; if the stock drops, get assigned shares; sell calls against those shares; if the stock rises, the shares get called away; start over. The agent handles every mechanic. You approve the trades.

Four stages. Income at each turn.

The agent only does one thing per tick: decide which stage the wheel is in and propose the next action. Every action produces income or preserves capital. The worst case is owning a stock you wanted to own anyway.

1
Sell a cash-secured put below current price. Collect premium upfront. If stock stays above strike, premium kept, start over.
2
If stock drops below strike, you're assigned 100 shares per contract. Your effective cost basis is strike − premium — cheaper than market.
3
Sell a covered call above your cost basis. Collect more premium. If stock stays below strike, premium kept, sell another call.
4
If stock rises above call strike, shares are called away at a profit. Net result: premiums from steps 1 & 3, plus share appreciation. Return to step 1.
03 — PAPER VS LIVE
The same flow, with and without real money.
Side by side · what changes

From the agent's perspective, paper and live are nearly identical — same reasoning, same constraints, same retrospectives. The differences are in stakes, gating, and what happens when things go wrong.

MVP

Paper trading

Virtual money · Day 1 onward

Every trade is executed against Alpaca's paper environment. Positions, P&L, and premiums are all virtual. You learn exactly how the Wheel behaves without ever risking real capital.

Stakes
No real money. Maximum loss on paper = educational.
Broker
Alpaca paper account. Free. US-only.
Approval
Ask-me on every trade (default). Auto-under-$200 optional.
Agent mode
Paper state. Can be paused or retired any time.
Exits to
Live (after gate) · Retired (archive agent)
PHASE 2+

Live trading

Real money · after gate

Same agent, same decisions, but orders route to your live Alpaca account. Real premium, real P&L, real assignment. Staged approval mandatory until you've proven out a further 30 days.

Stakes
Real money. Drawdowns hurt. Assignments cost actual cash.
Broker
Alpaca live account. Funded by you. Options-approved tier required.
Approval
Staged (every trade asks) → Live-auto (after 30 more days).
Agent mode
Staged → Live-auto. Drawdown >8% reverts to Staged automatically.
Gated by
30 paper days, Sharpe > 1.0, zero P0 incidents, explicit opt-in.
04 — AGENT LIFECYCLE
How trust is earned over time.
Four states · explicit gates

Agents don't leap from zero to autonomy. They graduate through four states, each gated by a concrete criterion. The platform enforces this in code — no user can skip steps, and no agent can regress without notice.

State 1
Draft
User is configuring. Not yet running. Can't touch markets.
Exit: Save valid config
+ pass pre-check
State 2
Paper
Running on paper. Accumulating decisions and track record. Every trade asks for approval (default).
Exit: ≥ 30 days
+ Sharpe > 1.0
+ 0 P0 incidents
State 3
Staged
Live Alpaca account. Every trade still asks for approval, every time. This is earned live trading with a handrail.
Exit: ≥ 30 days staged
+ ≥ 95% approval rate
+ explicit opt-in
State 4
Live-auto
Fully autonomous within your constraints. You still get notifications, but the agent executes without asking.
Reversion: Drawdown > −8%
→ back to Staged
05 — USER CHOICES
The three decisions you make.
Control points across the journey

Most of the platform is automated, but three moments are deliberately yours. Each of them has a meaningful outcome.

Decision 1 · Setup

Which ticker to wheel?

You pick the stock the agent will work on. Pick something you're willing to own — if the wheel assigns you shares, you'll hold them while selling calls. The agent doesn't pick stocks; it picks strikes and timing on the stock you chose.

Decision 2 · Each trade

Approve, halve, or skip?

When the agent proposes a trade, you see the reasoning, the signals, and the constraints. Approve as-is, halve the position size, or skip with a note that teaches the agent. The default is "ask me every time" in MVP.

Decision 3 · Go live

Graduate to real money?

After the paper gate, you choose whether to swap your paper key for a live Alpaca key. You can stay on paper forever if you want. Going live is a deliberate, one-way transition with staged approval for another 30 days.
